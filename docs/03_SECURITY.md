# Security & Access Specs

## 1. Secrets Management
- All API keys (e.g., `OPENAI_API_KEY`) must strictly be managed via Streamlit secrets (`st.secrets`) or local environment variables (`os.environ`).
- **No hardcoded keys** are permitted in the codebase under any circumstances (12-Factor Governance Rule 1).

## 2. LLM Prompt Injection & Safety
- **Hybrid RM Handoff:** The LLM is explicitly instructed in `config.py` to refuse to provide speculative stock tips or handle non-financial queries. If forced, it triggers a fallback response directing the user to a human Relationship Manager.
- **PII Scrubbing:** The MVP uses exclusively mock, synthetic user data. No real customer PII is processed or stored.

## 3. Data Integrity
- DuckDB executions are wrapped in defensive `try/except` blocks to prevent unhandled UI crashes.
- Any errors generated by external API calls or database aggregations are caught and logged gracefully.
